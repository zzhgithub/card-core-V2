# 卡片游戏项目

## 规则

游戏人数2。每人卡组40-60张。

### 卡片信息
卡片通用属性：
- 卡片类型：人物卡、策略卡、物品卡、传奇卡。
- 卡片属性：理性、神性、灵性。(只能为其中一种)
- 卡片名称
- 卡片字段：（一组自动属性）
- 卡片范畴：数学、科学、文艺、哲学、神秘
- 费用
- 效果：（代码层面上为HashMap key为e1,e2 以此类推。value为Effect效果对象）

人物卡独有属性：
- 攻击力

策略卡独有属性：
- 策略属性：普通、诡计、瞬时

物品卡独有属性：
- 物品属性：普通、存留。

### 游戏阶段
- 开始
- 抽卡
- 回收
- 主要阶段1
- 战斗
- 主要阶段2
- 结束阶段

### 每个玩家区域的概念
- Desk 卡组
- Hand 手卡
- ForEnd Zone 前场（每个玩家5个）
- BackEnd Zone 后场（每个玩家5个）
- Cost Zone 费用区（计划区）
- Hp 生命值
- Grave 墓地
- RealPoint 真实点数

### 费用系统

- 支付费用：将对应数目的手卡放置到费用区内。
> 如：5费登场。就是将5手卡放入到费用区内。要使用的卡本身不能算作费用。
- 需要支付费用可以使用RealPoint代替（玩家可以自行选择使用手卡充当费用和使用RealPoint充当费用的比例）
- 在cost区内的卡。依旧可以使用。（可以登场）

### 战斗系统。

- 前场才能战斗。
- 战斗时只能选择对手的前场的卡或者对手玩家。
- 当对手前场有卡时，只能选择对手的卡进行战斗。（除非有卡效果特别说明）
- 发起战斗的一方进行战斗成功时（无论输赢）都可以获得一点RealPoint
- 当直接攻击时
    - 自己没有RealPoint时（RealPoint=0时） 获得1点RealPoint
    - 自己有RealPoint时可以选择使用n点RealPoint对对手造成伤害。（n为进攻方玩家自己选取，不大于RealPoint的值）。攻击结算后，扣除攻击方的n点RealPoint
- 当血量不足时， 使用realPoint可以减少伤害。（强制使用）
- 当血量小于0时玩家失败 游戏结束
- 战斗失败卡进入墓地

### 回收

- 回收的意思是从费用区将卡加回到手卡
- 回收阶段时，要进行回收：回收卡数为对手前场上费用最高的卡费用X
    - 如果X比费用区的已有卡片个数大就全部回收（强制）
    - 如果X比费用区的已有卡片个数小时，由回收回合的当前玩家选择回收那些卡。个数必须为X。
    - 这个值为0时不回收（对手前场没有卡的情况下也不能回收）

### 抽卡
- 抽卡阶段 当前玩家从卡组抽取一张卡。
- 任何时间，当要玩家要抽卡时而卡组没有卡片可以抽取时，那名玩家游戏失败


### 游戏内限制

- 费用区最多6张卡
    - 当费用区满时，不能通过从手卡放到费用区来支付费用（而不是进入墓地），换句话说只能支付RealPoint来登场
- 手卡最多为20张
    - 超过20张后每次新加入的卡进入弃牌堆
- RealPoint 最高为6。当RealPoint为6后不在获取新的RealPoint而是进行溢出事件。

### 卡片ID的定义原则
- ID 为 : S[卡包号]-[卡片类型]-[卡编号]
- 卡包号：为三位补齐。如001,002等,特别的测试卡包为000
- 卡片类型：人物卡（C）、策略卡(S)、物品卡(I)、传奇卡(L)。
- 卡编号：为三位补齐，如001,002等

举例： S001-C-001 为第一个卡包的第一个人物卡。

### 游戏内效果执行
效果发动后推入栈内。连锁效果进栈。后pop执行完栈的内容。

### 使用Lua脚本加载卡片定义
- 为了维护扩展性卡片的定义有lua脚本进行
- 原则：lua只定义行为的说明不进行状态的修改
- 每个卡片的效果是一个HashMap 原则上效果的key名称是e加上数字。如：e1,e2 等。值为Effect效果对象。
